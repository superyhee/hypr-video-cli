# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

hypr-video-cli is a CLI tool that renders MP4 videos from canvasState JSON. It also ships as a Claude Code skill (`/install-skill ./skill`) that lets Claude generate videos from natural language descriptions.

## Commands

```bash
# Render a video from JSON
npx hypr-video-cli input.json -o output.mp4
node dist/render-video.mjs input.json -o output.mp4

# Validate canvasState JSON (catches common mistakes before rendering)
npx tsx scripts/validate-canvas-state.ts input.json

# npm scripts
npm run render -- input.json -o output.mp4
npm run validate -- input.json
```

Prerequisites: Node.js >= 20, FFmpeg in PATH.

## Architecture

This is a **pre-bundled CLI** — there is no `src/` directory or build step. The entire renderer is bundled into a single file:

- `dist/render-video.mjs` — the bundled renderer, compiled from an external monorepo. Do NOT edit this file directly.
- `bin/render-video` — entry point shim that dynamic-imports `dist/render-video.mjs`.
- `scripts/validate-canvas-state.ts` — standalone validator (zero project deps, runs with tsx). This is the only TypeScript source file you should edit.
- `skill/SKILL.md` — Claude Code skill definition (frontmatter + workflow).
- `skill/references/` — feature reference docs (elements, animations, transitions, keyframes, captions, effects, canvas-state, common-mistakes).
- `examples/` — 14 ready-to-render JSON templates for various video types.

### Rendering Pipeline

1. CLI parses input JSON (accepts either full payload with `canvasState` + `outputFormat`, or bare canvasState)
2. `DataConverter` transforms JSON into internal render model
3. Each frame is rendered onto a skia-canvas `Canvas` — elements drawn in track order, animations/effects applied per-frame
4. Frames piped to FFmpeg as raw image data to produce final MP4
5. sharp used for image processing (resizing, format conversion)

### Key Dependencies

- `skia-canvas` — Canvas 2D rendering (native binary, no browser needed)
- `sharp` — image processing (native binary)
- FFmpeg — video encoding (must be installed on system)

## canvasState JSON Structure

The input JSON has these critical fields: `width`, `height`, `backgroundColor`, `maxTime`, `elements`, `tracks`, `animations`, `captions`, `globalCaptionStyle`. Optional: `keyframeTracks`, `transitions`.

### Critical Gotchas (most common causes of broken output)

- Tracks MUST have `"isVisible": true` — omitting it renders the track invisible
- `opacity` goes at element top level, NOT inside `placement`
- Text elements use `fontColor` for color, NOT `fill` (fill is for shapes only)
- `maxTime` must be >= the largest element `timeFrame.end`
- Time units: `timeFrame`/animations/transitions use ms; captions use SRT format (`"HH:MM:SS.mmm"`); `wordTimings` use seconds (float); keyframe `time` is ms relative to element start

## Development Guidelines

**Editable files:** Only `scripts/validate-canvas-state.ts`, `skill/` contents, `examples/`, and `CLAUDE.md`. Never edit `dist/render-video.mjs` — it is compiled from an external monorepo.

**Output directory:** Rendered videos are saved to `output/` by convention (gitignored). Always use `-o ./output/<name>.mp4` when rendering.

**Adding examples:** Place new JSON files in `examples/`. Validate before committing: `npm run validate -- examples/<name>.json`.

## Troubleshooting

- **`FFmpeg not found`** — Install FFmpeg and ensure it's in PATH (`brew install ffmpeg` on macOS)
- **`skia-canvas` or `sharp` install fails** — These are native binaries. Ensure matching Node.js version (>=20) and platform. Try `npm rebuild`.
- **Blank video output** — Most likely `isVisible: true` is missing on tracks, or a full-screen shape element on the top track is occluding content. Run `npm run validate` to catch common issues.
- **Elements cut off early** — Check that `maxTime` >= the largest element `timeFrame.end`.
